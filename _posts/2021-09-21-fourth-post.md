@ 다음은 사용자가 로그인 셸을 변경하는 과정이다. ( 괄호 ) 안에 들어갈 옵션으로 알맞은 것은?

2.	-l

[chsh = change shell]

1. -c : 없는 옵션입니다.
2. -l : (list) chsh -l을 입력했을 경우 변경 가능한 쉘을 출력합니다.
3. -s : (shell) 로그인 쉘을 사용자가 입력한 쉘로 변경합니다.
4. -u : (help) chsh 명령어에 관한 도움말을 출력합니다.

@ 다음 설명과 관련 있는 명령으로 알맞은 것은?
- 오랜 시간이 소요되는 백업 명령을 실행하려고 한다. 
작업 중인 터미널이 닫혀도 계속 실행될 수 있게 명령을 내리려고 한다.
1.	nice
2.	renice
3.	nohup
4.	bg

nice: 프로세스의 우선순위를 변경하는 명령어. 기본 NI값(0)을 -20 ~ 19까지의 값으로 증가나 감소한다. 작을수록 우선순위가 높다.

renice: 프로세스의 우선순위를 변경하는 명령어. nice는 기존의 NI값에 증가나 감소하는 형태지만, renice는 기존의 NI 값에 상관없이 지정한 NI값이 바로 설정된다.

nohup: 사용자가 로그아웃하거나 작업 중인 터미널창이 닫혀도 실행 중인 프로세스를 백그라운드 프로세스로 작업될 수 있도록 해주는 명령어. 사용자가 명령행 뒤에는 '&'을 써야 한다.

bg: fore ground 프로세스를 back ground 프로세스로 전환하는 명령어

1.	IP
2.	VRRP

[TCP]
.프로토콜 연결 수립
- 연결지향형
- 3-way handshaking 연결 방식
.신뢰성
- 전송을 신뢰할 수 있다.
.전송속도
- 빠르지만 UDP보다 느림
. 부하
- 낮지만 UDP보다 높은
. 기타
- 패킷들의 전송흐름을 제어

[UDP]
.프로토콜 연결 수립
- 비 연결형
.신뢰성
- 전송을 신뢰할 수 없다.
.전송속도
- 매우 빠르다
. 부하
- 매우 낮음
. 기타
- IP

[IP]
- 인터넷을 구성하는 핵심요소
- 인터넷 전송단위인 Datagram 정의
- 인터넷 주소체계 정의
- 네트워크 계층과 전송 계층 사이의 데이터 이동

[VRRP]
(Virtual Redundancy Router Protocol)
- VRRP는 RFC Standard 문서에 정의된 표준입니다.
  HSRP와 다르게 Cisco 뿐만 아니라, 모든 Vendor에서도 지원되는 Protocol 입니다.
